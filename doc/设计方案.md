# Novel2Screen Agent Engineering Specification v2.0
## AI-Ready Edition

> Multi-Agent System for Novel-to-Screenplay Conversion  
> Supports 3+ chapters, short/long modes, human edit round-trip

---

## 1. System Overview

**Input**: Novel text (minimum 3 chapters)  
**Output**: Structured screenplay in YAML format  
**Modes**: 
- `short` (3–10 chapters, faster, fewer agents)
- `long` (>10 chapters, full agent chain)

---

## 2. Core Architecture

### 2.1 Workflow Graph (LangGraph)

```text
Upload → Parser → Segmenter → ModeRouter
  ↓
NarrativeAgent → CharacterAgent → WorldAgent(optional) → TimelineAgent(simplified for short)
  ↓
EpisodePlanner → ScenePlanner → DialogueWriter → YAMLCompiler
  ↓
CriticAgent → RepairAgent → Export
```



### 2.2 ModeRouter Logic

python

```
def route_mode(num_chapters: int) -> str:
    return "short" if 3 <= num_chapters <= 10 else "long"
```



**Short mode behavior**:

- WorldAgent: skip, use default template
- TimelineAgent: linear only, no conflict detection
- Retrieval top_k: 3 (instead of 5)

------

## 3. Agent Specifications

Each agent implements:

python

```
def run(input: dict) -> dict   # strict output per schema
def validate(output: dict) -> bool
def retry(max_attempts=2) -> dict
```



### 3.1 NarrativeAgent

**Input**: `{"chunks": list[str]}`
**Output**: `{"major_events": list[dict], "subplots": list[dict], "turning_points": list[dict], "theme": str}`
**Success**: ≥95% major event coverage

### 3.2 CharacterAgent

**Output schema**:

json

```
{
  "characters": [{
    "id": "char_001",
    "name": "string",
    "role": "protagonist|antagonist|supporting",
    "goal": "string",
    "fear": "string",
    "arc": "string",
    "voice_style": "string"
  }]
}
```



### 3.3 WorldAgent (long mode only)

**Output schema**:

yaml

```
world_rules:
  magic: optional
  technology: optional
  politics: optional
geography: list
```



### 3.4 TimelineAgent

**Short mode**: output `{"events": [{"chapter": int, "description": str}]}` (linear)
**Long mode**: output `{"graph": {"nodes": [], "edges": []}, "conflicts": []}`

### 3.5 ScenePlannerAgent

**Scene contract**:

json

```
{
  "scene_id": "sc_<num>",
  "location": "string",
  "time": "string",
  "objective": "string",
  "conflict": "string",
  "emotion": "string"
}
```



### 3.6 DialogueAgent

**Rules**:

1. Each dialogue line must match character's `voice_style`
2. Dialogue must advance plot or reveal character
3. No exposition dumps (max 3 lines of uninterrupted exposition per scene)

### 3.7 CriticAgent

**Checks** (return list of violations):

- continuity (reference to previous scenes)
- pacing (scenes per episode ≤12)
- character motivation (goal/arc alignment)
- dialogue quality (no repetitive filler)
- shootability (no internal thoughts as action)
- line balance (optional: Gini coefficient <0.6)

### 3.8 RepairAgent

**Auto-fix capabilities**:

- timeline conflicts → reorder events
- duplicate scenes → merge or delete
- character drift → rewrite dialogue using character bible
- schema violations → add missing fields with defaults
- missing visual/sound → infer from context or insert null

### 3.9 BidirectionalConsistencyAgent (optional)

**Input**: `original_novel_chunks`, `human_edited_yaml`
**Output**: `{"alignment_score": float, "deviations": list, "suggestions": list}`

------

## 4. Memory Architecture

### 4.1 Short-Term Memory

- Active chapter (current chunk)
- Active scene (in generation)
- Last 5 dialogue turns
- TTL: session only

### 4.2 Long-Term Memory

- Character bible (persistent across tasks)
- World bible
- Relationship graph

### 4.3 Semantic Memory (Vector DB)

- Embedding model: `BAAI/bge-large-en-v1.5`
- Chunk size: 1500 tokens
- Overlap: 200 tokens
- Top-K retrieval: 5 (long mode) / 3 (short mode)

## 5. YAML Schema (Final)

### 5.1 Root Structure

yaml

```
title: string
logline: string
genre: string
theme: string
characters:
  - id: string
    name: string
    role: string          # protagonist/antagonist/supporting
    goal: string
    arc: string
episodes:
  - id: string            # ep_001
    title: string
    summary: string
    scenes: []
```



### 5.2 Scene Structure

yaml

```
scene_id: string
location: string
time: string
visual_focus: string | null      # what camera sees
sound_effect: string | null      # diegetic or non-diegetic
voice_over: string | null        # narration text

beats:                           # sequence of micro-actions
  - type: string                 # dialogue | action | silence | reaction
    character_id: string | null  # required if type=dialogue
    content: string              # line or description
    emotion: string | null       # anger, fear, joy, sadness, tension

transition: string               # cut | fade | dissolve | wipe
duration_estimate: string        # e.g., "45s", "2m"
```



### 5.3 Example YAML

yaml

```
title: "The Last Query"
logline: "A detective confronts her past in an abandoned server farm."
genre: "cyberpunk noir"
theme: "memory and guilt"
characters:
  - id: char_01
    name: "Kaelen"
    role: "protagonist"
    goal: "Find the truth about her partner's death"
    arc: "from guilt to acceptance"
episodes:
  - id: ep_01
    title: "Ghost in the Wire"
    summary: "Kaelen receives a mysterious message."
    scenes:
      - scene_id: sc_01
        location: "abandoned server farm"
        time: "night"
        visual_focus: "flickering LEDs on dead servers"
        sound_effect: "distant electrical hum"
        voice_over: null
        beats:
          - type: "action"
            character_id: null
            content: "Kaelen wipes dust off a terminal screen."
            emotion: null
          - type: "dialogue"
            character_id: "char_01"
            content: "You're not real. You can't be."
            emotion: "fear"
          - type: "silence"
            character_id: null
            content: "The screen glitches. Static for 3 seconds."
            emotion: "tension"
          - type: "reaction"
            character_id: "char_01"
            content: "Kaelen steps back, hand on her weapon."
            emotion: "alert"
        transition: "cut"
        duration_estimate: "50s"
```

## 6. Human-in-the-Loop Edit Workflow

### 6.1 Export

Endpoint: `GET /export/yaml?task_id={id}`
Returns: YAML file (as above)

### 6.2 Import Edits

Endpoint: `POST /import-edits`
Body: YAML file (user-modified)

**Processing steps**:

1. Validate YAML against schema (return 422 on failure)
2. Compute diff: original vs edited (field-level)
3. Run CriticAgent only on changed scenes/episodes
4. Run RepairAgent in incremental mode (fix only violations in changed parts)
5. (Optional) Run BidirectionalConsistencyAgent to produce alignment report
6. Return final reconciled YAML

### 6.3 Overlay Rules

- If user adds a new scene: accepted as is, no automatic modification
- If user deletes a scene: removed from output
- If user changes `character_id` in a beat: system logs a warning but keeps change
- If user modifies `visual_focus` or `sound_effect`: always accepted

## 7. Prompt Engineering Standard

Every prompt must contain the following sections **exactly**:

text

```
ROLE: <agent role>
MISSION: <specific task>
INPUT: <JSON schema of input>
CONSTRAINTS:
- <rule 1>
- <rule 2>
OUTPUT_SCHEMA: <JSON schema>
EXAMPLES:
- input: {...}
  output: {...}
FAILURE_CONDITIONS:
- <condition that causes retry>
```



**Example for ScenePlannerAgent**:

text

```
ROLE: Scene Planner Agent
MISSION: Convert a story episode into a sequence of scene contracts.
INPUT: {"episode_id": "string", "summary": "string", "characters": [{"id": "string", "goal": "string"}]}
CONSTRAINTS:
- Each scene must have exactly one objective and one conflict.
- Minimum 2 scenes per episode, maximum 12.
- Scenes must follow chronological order.
OUTPUT_SCHEMA: {"scenes": [{"scene_id": "string", "location": "string", "time": "string", "objective": "string", "conflict": "string", "emotion": "string"}]}
EXAMPLES:
- input: {"episode_id": "ep_01", "summary": "Hero enters the cave and finds the sword.", ...}
  output: {"scenes": [{"scene_id": "sc_01", "location": "cave entrance", ...}]}
FAILURE_CONDITIONS: Output has fewer than 2 scenes, or any scene missing objective.
```

## 9. Database Schema

Tables (PostgreSQL):

sql

```
CREATE TABLE novels (
    id UUID PRIMARY KEY,
    title TEXT,
    raw_text TEXT,
    chunk_count INT
);

CREATE TABLE tasks (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    mode TEXT CHECK (mode IN ('short', 'long')),
    status TEXT,
    created_at TIMESTAMP
);

CREATE TABLE human_edits (
    id UUID PRIMARY KEY,
    task_id UUID REFERENCES tasks(id),
    original_yaml_hash TEXT,
    edited_yaml TEXT,
    reconcile_status TEXT
);

CREATE TABLE exports (
    id UUID PRIMARY KEY,
    task_id UUID REFERENCES tasks(id),
    yaml_content TEXT,
    exported_at TIMESTAMP
);
```



Indexes: `novel_id`, `task_id`, `reconcile_status`
